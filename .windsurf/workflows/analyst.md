# Business & Product Analysis Workflow

When this workflow is triggered, perform a comprehensive business and product analysis of the codebase.

## Step 1: Project Overview
- Identify the project type (SaaS, e-commerce, internal tool, etc.)
- Determine the target audience/users
- List the core value propositions

## Step 2: Feature Inventory
- Catalog all implemented features
- Identify features that are partially complete
- Note any placeholder or mock implementations
- List features referenced but not yet built

## Step 3: Monetization Analysis
- Identify existing payment/subscription integrations
- Review pricing models if present
- Analyze usage tracking mechanisms
- Check for premium/gated features

## Step 4: User Journey Mapping
- Map the authentication flow
- Identify onboarding experience
- Review key user workflows
- Note friction points or missing steps

## Step 5: Competitive Positioning
- Based on features, suggest market positioning
- Identify unique differentiators
- Note common SaaS features that may be missing

## Step 6: Technical Debt Impact on Business
- Identify technical issues affecting user experience
- Note performance concerns that impact conversion
- List security gaps that could affect trust

## Step 7: Growth Readiness
- Evaluate scalability of current architecture
- Check for analytics/tracking implementation
- Review SEO readiness
- Assess mobile responsiveness

## Step 8: Recommendations
Provide prioritized recommendations in three categories:
1. **Quick Wins** - Low effort, high impact improvements
2. **Strategic Investments** - Medium effort features for growth
3. **Technical Foundation** - Infrastructure improvements needed

## Output Format
Present findings in a structured report with:
- Executive Summary (2-3 sentences)
- Key Metrics (feature count, completion %, etc.)
- Detailed Analysis by Section
- Prioritized Action Items
